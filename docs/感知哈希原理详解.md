# 感知哈希（pHash）原理详解

> 本文档从三个角度讲解感知哈希：技术实现、儿童理解、统计学视角

---

## 一、什么是感知哈希

传统哈希（MD5/SHA）的特点是：**输入哪怕只有 1bit 变化，输出也会完全不同**。

感知哈希则不同：**图片内容视觉上相似 → 哈希值也相似**。

| 特性 | 传统哈希 | 感知哈希 |
|-----|---------|---------|
| 目标 | 数据完整性 | 内容相似度 |
| 1bit 变化 | 输出完全不同 | 输出变化小 |
| 相似图片 | 哈希完全不同 | 哈希相似 |
| 用途 | 校验文件 | 找相似图/去重 |

## 二、计算步骤详解

### 步骤 1：缩放到 32×32

```javascript
const canvas = new OffscreenCanvas(32, 32)
const ctx = canvas.getContext('2d')
ctx.drawImage(image, 0, 0, 32, 32)  // 把大图缩小画上去
```

**目的**：消除尺寸差异，所有图片统一成 32×32 小图。

**为什么先 32×32 而不是直接 8×8？**
- 直接 8×8：从原图只采 64 个点，太稀疏，易误判
- 先 32×32：采 1024 个平均值，信息更完整，再浓缩到 8×8

---

### 步骤 2：转灰度

```javascript
for (let i = 0; i < imageData.data.length; i += 4) {
  const r = imageData.data[i]
  const g = imageData.data[i + 1]
  const b = imageData.data[i + 2]
  gray[i / 4] = 0.299 * r + 0.587 * g + 0.114 * b
}
```

**目的**：去掉颜色干扰，只保留明暗信息。

---

### 步骤 3：8×8 分块（简化 DCT）

```javascript
const blocks = new Float32Array(8 * 8)

for (let by = 0; by < 8; by++) {      // 8 行块
  for (let bx = 0; bx < 8; bx++) {    // 8 列块
    let sum = 0
    
    // 每个块是 4×4 = 16 个像素
    for (let y = 0; y < 4; y++) {
      for (let x = 0; x < 4; x++) {
        const py = by * 4 + y
        const px = bx * 4 + x
        sum += gray[py * 32 + px]
      }
    }
    
    blocks[by * 8 + bx] = sum / 16    // 存平均值
  }
}
```

**可视化**：
```
32×32 灰度图:
┌───┬───┬───┬───┬───┬───┬───┬───┐
│ 0 │ 1 │ 2 │ 3 │...              │  ← 第 0 行块 (4×4像素)
├───┼───┼───┼───┼───┼───┼───┼───┤
│ 8 │ 9 │...                      │  ← 第 1 行块
├───┼───┼───┼───┼───┼───┼───┼───┤
... 共 8×8 = 64 个块
```

**为什么用 8×8 而不是直接用 32×32？**
- 32×32 = 看每个人的头发丝、衣服褶皱（太敏感）
- 8×8 = 只看谁站第一排、谁站最后一排（抓大放小）
- 抗干扰：压缩、裁剪、亮度调整都不会改变整体结构

---

### 步骤 4：生成哈希

```javascript
const avg = blocks.reduce((a, b) => a + b, 0) / 64

let hash = ''
for (let i = 0; i < 64; i++) {
  hash += blocks[i] > avg ? '1' : '0'
}

return hash  // 64位字符串，如 "101101001011..."
```

**作用**：把 64 个亮度值变成 64 个 0/1，比平均亮→1，暗→0。

---

### 步骤 5：汉明距离比较

```javascript
hammingDistance(hash1, hash2) {
  let distance = 0
  for (let i = 0; i < 64; i++) {
    if (hash1[i] !== hash2[i]) distance++
  }
  return distance  // 0-64，越小越相似
}
```

**示例**：
```
hash1: 1011010110...
hash2: 1010010110...
           ↑ 这里不同
distance = 1
相似度 = 1 - 1/64 = 98.4%
```

**经验阈值**：
- 汉明距离 ≤ 5：非常相似（可能是同一张图）
- 汉明距离 ≤ 10：相似图片（变体/编辑版）
- 汉明距离 > 20：不同图片

## 三、儿童版讲解

### 比喻 1：「影子游戏」

> 晚上用手电筒照你的玩具：
> - 玩具离墙近 → 影子小
> - 玩具离墙远 → 影子大
> - 玩具转个圈 → 影子形状变了
>
> 但**小汽车的影子始终是小汽车**，不会变成小狗。
>
> 感知哈希就是这个影子——不管大小，形状对了就行。

### 比喻 2：「看全班照片」

> 你站在教室后面，看全班同学的照片墙：
>
> **32×32 = 看每个人的头发丝、衣服褶皱**
> - 小明今天换了件衬衫 → 完全不一样了！
>
> **8×8 = 只看谁站第一排、谁站最后一排**
> - 小明换衬衫？还是站那个位置 → 一样！

### 计算步骤顺口溜

> **"大变小，彩变灰，分格子，比明暗，得密码"**

### 一句话记住

> **"长得像的东西，指纹也像；不像的东西，指纹就不像。"**

## 四、统计学视角

### 各步骤对应的统计概念

| 步骤 | 统计概念 |
|-----|---------|
| 32×32 缩放 | **抽样** - 从总体中抽取代表性样本 |
| 4×4 平均 → 8×8 | **分层抽样 + 均值估计** |
| 与平均值比较 | **二值化** - 把连续变量变分类变量 |
| 汉明距离 | **差异度量** |
| 相似度阈值 | **假设检验** - 判断是否"同一类" |

### 核心统计思想

```
原图 = 总体（几百万像素）
    ↓
32×32 = 大样本（1024个观测值）
    ↓
8×8 块平均 = 降维 + 去噪（平滑局部波动）
    ↓
与总平均比较 = 标准化（消除亮度绝对值影响）
    ↓
0/1 哈希 = 非参数化（不假设分布）
```

**关键洞察**：
> 两张图是否相似，不取决于绝对亮度，而取决于**明暗的相对模式**（哪些区域亮、哪些暗）

这正是统计学里**标准化**和**分布形态**的思想。

## 五、信号处理视角

```
图片 = 二维信号
    ↓
DCT（离散余弦变换）= 把信号分解为不同频率
    ↓
低频 = 整体结构（保留）
高频 = 细节噪声（丢弃）
    ↓
8×8 = 只保留最低频的 64 个系数
```

代码里的简化版：
```javascript
// 4×4 平均 ≈ 低通滤波器（去掉高频噪声）
sum / 16  // 平均就是最简单的平滑滤波
```

## 六、简化版 vs 标准 pHash

| | 标准 pHash | 本代码（简化版） |
|--|-----------|---------------|
| DCT 变换 | 完整数学公式 | 4×4 平均 |
| 精度 | 高 | 够用 |
| 速度 | 慢 | 快 |
| 适用场景 | 学术/高精度 | 前端实时去重 |

**简化原理**：4×4 平均 ≈ 低频分量，足够判断相似度。

---

## 七、一句话总结

> **感知哈希 = 用统计学方法（抽样、平均、标准化）+ 信号处理方法（降维、滤波）来提取图片的「形态特征」**

它不仅是个算法，更是一套**降维 + 特征工程**的完整方法论。
